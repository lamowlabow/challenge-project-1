Semantic HTML - A Cheat Sheet

“Do you wish me a good morning, or mean that it is a good morning whether I want it or not; or that you feel good this morning; or that it is a morning to be good on?”

Semantic HTML refers to the visual meaning behind the elements we choose to add.

Before it was introduced, tags didn't do a great job in conveying what they were used for.

Semantic Tags

Tag Name Description
<header> Header A container for introductory elements that typically go on top of the page.
<nav> Navigation A section of a page with a purpose of providing navigational links.
<main> Main Represents the main content section of the body element.
<footer> Footer Content reserved for the bottom of a page or section.
<video> Video Embeds a media player which supports video playback into the document.
<audio> Audio Embeds an audio player which supports audio playback into the document.
<embed> Embed Embeds external content at the specified point in the document.
<figure> Figure A container for self-contained content, like illustrations and photos.
<figcaption> Figure Caption A caption that is made for a <figure> element
<section> Section Defines a standalone section that is contained within a document.
<article> Article Defines self-contained content, like an article or a blog post.
<aside> Aside An element used to define the content it is referencing.
